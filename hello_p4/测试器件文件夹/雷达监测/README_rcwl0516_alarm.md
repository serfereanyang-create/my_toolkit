# RCWL-0516 + ESP32-P4 + 低电平触发蜂鸣器 复现说明 
 
本项目使用 [`RCWL-0516`] 微波雷达模块检测人体移动，配合 [`ESP32-P4`] 开发板和三引脚低电平触发蜂鸣器，实现“人离开后报警”。 
 
## 1. 功能说明 
 
当前版本逻辑： 
 
- 检测到有人移动时，不报警，只记录“有人来过” 
- 当雷达输出从高电平恢复为低电平时，认为“人离开/移动结束” 
- 蜂鸣器报警约 3 秒后停止 
 
注意：[`RCWL-0516`] 是移动检测模块，不是人体存在检测模块，因此这里的“离开”本质是“移动结束”。 
 
## 2. 所需硬件 
 
- 1 x [`ESP32-P4`] / 开发板 
- 1 x [`RCWL-0516`] 微波雷达模块 
- 1 x 三引脚低电平触发有源蜂鸣器模块 
- 若干杜邦线 
- USB 数据线 
 
## 3. 接线说明 
 
推荐接线： 
 
- 雷达 [`VIN`] - [`5V`] 
- 雷达 [`GND`] - [`GND`] 
- 雷达 [`OUT`] - [`GPIO1`] 
- 雷达 [`CDS`] - 
- 蜂鸣器 [`VCC`] - [`3V3`]（若模块要求 5V，则接 [`5V`]） 
- 蜂鸣器 [`GND`] - [`GND`] 
- 蜂鸣器 [`IN`] / [`SIG`] - [`GPIO5`] 
 
重点：所有设备必须共地，也就是所有 [`GND`] 要连接在一起。 
 
## 4. 代码文件 
 
Arduino 草图文件：[`rcwl0516_alarm_esp32p4.ino`] 
 
默认引脚定义： 
 
```cpp 
const int RADAR_PIN = 1; 
const int BUZZER_PIN = 5; 
``` 
 
如果你的接线不同，请同步修改代码中的 GPIO 定义。 
 
## 5. 烧录步骤 
 
1. 用 Arduino IDE 打开 [`rcwl0516_alarm_esp32p4.ino`] 
2. 选择正确的开发板型号和串口 
3. 点击编译并上传 
4. 打开串口监视器，波特率设为 [`115200`] 
5. 在雷达感应区内移动进行测试 
 
## 6. 现象判断 
 
正常现象： 
 
- 人进入并移动时，串口会检测到运动 
- 人停止移动并离开感应后，蜂鸣器响 3 秒 
 
异常排查： 
 
- 蜂鸣器一直响：说明模块可能不是低电平触发，或者高低电平逻辑相反 
- 完全不响：检查 [`GPIO5`] 接线、蜂鸣器供电、电源地是否共地 
- 雷达没反应：检查 [`VIN`] 是否接 5V，[`OUT`] 是否接到 [`GPIO1`] 
- 误触发较多：调整安装位置，远离大面积金属、风扇、电源噪声源 
 
## 7. 关于 [`CDS`] 引脚 
 
[`CDS`] 是光控/禁止检测辅助脚： 
 
- 小于约 0.7V：模块停止检测 
- 大于约 3.3V：模块正常工作 
 
当前复现阶段建议 [`CDS`] 悬空，先保证主功能跑通。
